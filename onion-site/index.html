Verify before you install

Never install an APK you have not verified. After downloading, compute the hash of your copy and confirm it matches the published checksum: 

sha256sum {{.APKName}}

Or verify against the list of checksums in one step: 

curl -O http://{{.OnionAddress}}/{{.APKName}}
curl -O http://{{.OnionAddress}}/SHA256SUMS
sha256sum -c SHA256SUMS

Expected output: {{.APKName}}: OK. If the hashes differ, delete the file and do not install it. The authoritative list is at /SHA256SUMS.

Installing

  1. Download the .apk onto your Android device (or transfer it across).
  2. Verify its SHA-256 matches /SHA256SUMS as above.
  3. Open the file. Android will ask to allow installs from your browser or file manager — grant Install unknown apps for that app (Settings → Apps → Special access → Install unknown apps).
  4. Confirm the install. Play Protect may warn you because the app is sideloaded.
  5. To update, download a newer build and install over the top — verify its checksum each time.

Don't trust us — self-host

The server is a Go binary, a PostgreSQL database, and a Docker Compose file. Run your own relay, including this very onion mirror: 

git clone https://github.com/jackofall1232/sublemonable.git
cd sublemonable
cp server/.env.example .env            # set DATABASE_URL, JWT key paths

# JWT signing keys (mounted into the server container)
mkdir -p server/keys
openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 \
  -out server/keys/jwt.pem
openssl rsa -in server/keys/jwt.pem -pubout -out server/keys/jwt.pub.pem

# Stage the release APK + checksums into the mirror (not committed to git):
cp sublemonable-v1.5.0-beta.apk onion-site/
sha256sum onion-site/*.apk > onion-site/SHA256SUMS

# Server + Postgres
docker compose up -d

# Add the Tor hidden service (this mirror is served from it):
docker compose -f docker-compose.yml -f docker-compose.tor.yml up -d

# Read your own .onion address:
docker compose exec tor cat /var/lib/tor/sublemonable/hostname

Then set TOR_ENABLED=true and ONION_ADDRESS (the hostname you just read) in .env and restart the server so it advertises the onion address to clients and serves this mirror over Tor.

The mirror content lives in onion-site/ and is mounted read-only into the server, which serves it at the root of the hidden service — and only there. Full instructions — TLS, certificate pinning, environment variables, APK staging, backups — are in the self-hosting guide.
